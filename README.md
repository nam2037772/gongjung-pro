# 공정표프로

엑셀 공종별 내역서를 업로드하면 입력한 공사기간을 기준으로 **CPM(Critical Path Method) 예정공정표 초안**을 자동 생성하는 정적 웹앱입니다.

전문 공정관리 프로그램(공정마당, MS Project 등)을 대체하는 도구가 아니라, 현장소장·공무담당자가 제출용 예정공정표 **초안을 빠르게 만들고 직접 다듬을 수 있는 실무형 작업대**입니다. 서버 저장 없이 브라우저에서만 동작하며, 결과물은 사용자가 직접 다운로드합니다.

"사진대지프로", "공사일보프로"와 같은 계열의 정적 웹앱으로, 빌드 도구 없이 `index.html`을 열거나 GitHub Pages 등 정적 호스팅에 올리기만 하면 바로 동작합니다.

## 사용법

1. **① 엑셀 업로드**: 공종별 내역서(.xlsx/.xls)를 업로드하거나 상단 "샘플 내역서로 테스트" 버튼으로 바로 체험합니다. 품명/규격/단위/수량/단가/금액 컬럼을 자동 인식하며, 인식이 애매하면 컬럼 매핑 화면이 나타납니다. "공종 자동분류" 버튼을 누르면 다음 단계로 이동합니다.
2. **② 공사정보 입력**: 공사명/현장명/발주처/시공사, 공사 시작일·종료일, 작업 주기(주5/6/7일), 휴무일을 입력하고 "공정표 생성"을 누릅니다.
3. **③ 공종/보할 편집**: 자동분류된 공종의 이름 수정, 병합, 삭제, 순서(order)/구분(순차·병행) 조정과 "보할 재계산"이 가능합니다.
4. **④ CPM 데이터**: 공종별 Activity(ID/기간/선행·후행/ES·EF·LS·LF/Float/주공정 여부)를 표 형태로 확인합니다.
5. **⑤ CPM 네트워크**: Activity를 노드로, 선행→후행 관계를 화살표로 표시하는 네트워크 다이어그램입니다. 주공정선(Critical Path)은 빨간색으로 강조되며, 노드를 클릭하면 하단에 상세정보(보할/기간/일정/선후행/ES·EF·LS·LF·Float)가 표시됩니다. 마우스 휠 확대·축소, 드래그 이동, 확대/축소/원래크기 버튼을 지원합니다.
6. **⑥ 예정공정표**: 일/주/월 단위가 자동 전환되는 Gantt 차트와 월별·주별 계획공정률, 누계 S-Curve를 확인합니다.
7. **⑦ 출력/다운로드**: 엑셀/CSV/JSON 다운로드, 인쇄(PDF 저장)를 실행합니다. JSON은 "작업 저장/불러오기" 용도로도 사용할 수 있습니다.

## 샘플 데이터

- 상단 **"샘플 내역서로 테스트"** 버튼: `js/sampleData.js`에 내장된 소규모 근린생활시설 신축공사 가상 내역(20개 공종, 평면 내역서 형식)을 즉시 불러옵니다.
- `sample/샘플공종별내역.xlsx`: 실제 신축공사의 계층형 "공종별집계표"(공종코드·상위공종·공종레벨 포함) 예시 파일입니다. 업로드 탭에서 이 파일로도 테스트할 수 있습니다.

## 주요 로직 설명

### 엑셀 파싱 (`js/utils/excelIO.js`)
- 상위 15개 행을 스캔해 품명/규격/단위/수량/단가/금액 등 키워드 점수가 가장 높은 행을 헤더로 추정합니다.
- "재료비/노무비/경비/합계" + "단가/금액" 2행 헤더 패턴(일반적인 산출내역서 양식)을 감지해 **"합계" 그룹의 "금액"** 컬럼을 최종 금액으로 사용합니다.
- 자동 인식이 불확실하면(공종명 또는 금액 컬럼 미검출) 컬럼 매핑 화면을 통해 사용자가 직접 지정합니다.
- "[ 합계 ]", "총계" 등 집계표 맨 아래 총괄행은 이중계상을 막기 위해 데이터 추출 단계에서 제외합니다.

### 공종 자동분류 (`js/core/classify.js`)
- 계층형 집계표(공종코드/상위공종 포함)는 **리크(leaf, 최말단 항목) + 상위 공종명 순차 매칭** 방식으로 분류합니다. 리크 자신의 이름이 표준 공종 키워드와 매칭되지 않으면 부모→조부모 순으로 올라가며 매칭을 시도합니다. 이렇게 하면 금액이 이중으로 집계되지 않으면서도 세부 품목이 올바른 상위 공종으로 귀속됩니다.
- 평면 내역서(계층 정보 없음)는 각 행을 리크로 간주해 자기 이름으로 바로 분류합니다.
- 표준 공종 20종에 대해 키워드 사전과 "선후관계 순서값(order)", "순차(chain)/병행(parallel) 구분(lane)"을 미리 정의해 두었습니다.

### 보할 계산 (`js/core/ratio.js`)
- 공종별 금액 ÷ 총 공사비 × 100으로 보할을 계산하고, 반올림 오차는 금액이 가장 큰 공종에서 자동 보정하여 합계가 정확히 100%가 되도록 합니다.

### 공정 기간 자동배분 (`js/core/schedule.js`)
- 공종을 "순차(chain)"와 "병행(parallel)" 두 레인으로 나눕니다.
- 순차 공종은 `가설→토공→지정/기초→골조→조적·금속·창호(병행 서브그룹)→방수→미장→타일→수장→도장→조경→기구/마감→준공청소` 순서로 슬롯을 구성하고, 슬롯 내 금액 비중에 비례해 근무일수를 배분한 뒤 앞뒤로 이어 붙입니다(끝단 슬롯에서 반올림 오차 보정).
- 전기·설비·통신·소방 등 병행 공종은 "골조(철근콘크리트공사) 종료 후 ~ 도장 시작 전" 구간 전체에 겹쳐 배치되며, 선행/후행 관계는 각각 그 구간 앞뒤의 순차 공종으로 연결됩니다.
- 근무일수가 공종 수보다 적으면 `PERIOD_TOO_SHORT` 오류를 반환해 UI가 경고를 표시합니다.

### CPM 계산 (`js/core/cpm.js`)
- schedule.js가 만든 선후관계 그래프에 대해 위상정렬(Kahn 알고리즘)로 순환참조를 탐지합니다(순환 시 `CYCLE_DETECTED`).
- 근무일 오프셋 기준으로 전진계산(ES/EF), 후진계산(LS/LF), Float(=LS-ES)를 계산하고 Float가 0인 Activity를 주공정선(Critical Path)으로 표시합니다.
- 최종적으로 오프셋을 근무일 캘린더(주5/6/7일 + 휴무일 반영)로 변환해 실제 시작일/종료일을 확정합니다.

### S-Curve (`js/core/scurve.js`)
- 각 Activity의 보할을 자기 기간의 근무일수로 균등 분배한 뒤, 일/주/월 구간별로 합산해 기간별 계획공정률과 누계(S-Curve)를 계산합니다.

### CPM 네트워크 다이어그램 (`js/ui/networkView.js`)
- cpm.js가 계산한 것과 **동일한 Activity 배열**을 그대로 사용해 Gantt/네트워크 간 Critical Path 판정이 항상 일치합니다.
- 각 Activity의 ES(Earliest Start) 값을 기준으로 열(column)을 구성해 좌→우 흐름으로 배치하고, 같은 ES를 가진 병행 Activity는 같은 열에 세로로 나열합니다.
- 선행→후행 관계는 SVG 베지어 곡선 화살표로 표시하며, `pred.critical && act.critical && pred.EF === act.ES`(여유 없이 맞물리는 구간)인 경우에만 빨간색 Critical 화살표로 강조해 실제 주공정 경로만 표시합니다.
- 노드를 클릭하면 하단 패널에 보할/기간/시작·종료일/선행·후행/ES·EF·LS·LF/Float/Critical 여부를 표시합니다.
- 마우스 휠·드래그·버튼으로 확대/축소·이동이 가능하며(순수 CSS transform, 외부 라이브러리 없음), 순환참조(CYCLE_DETECTED) 시에는 다이어그램 대신 오류 메시지를 표시합니다.

> **왜 React Flow를 쓰지 않았나요?** React Flow는 번들러(예: Vite) 기반 React 프로젝트를 전제로 하며 공식 CDN/UMD 배포가 없습니다. 이 프로젝트는 사진대지프로·공사일보프로와 동일하게 **빌드 도구 없는 정적 웹앱**으로 유지하는 것이 목표라서, 이미 S-Curve에 사용한 것과 같은 방식(순수 SVG + 소량의 이벤트 핸들러)으로 네트워크 다이어그램을 직접 구현했습니다. 덕분에 새 의존성 없이 GitHub Pages/Vercel에 그대로 배포할 수 있습니다.

## 기술 스택

- 순수 HTML/CSS/JavaScript(ES Modules), 빌드 도구 없음 — 정적 호스팅(GitHub Pages 등)에 그대로 배포 가능
- 엑셀 파싱·내보내기: [SheetJS(xlsx)](https://cdn.jsdelivr.net/npm/xlsx) CDN
- 날짜 계산: 외부 라이브러리 없이 `js/utils/date.js`에서 근무일 캘린더를 직접 구현
- 차트/다이어그램: 별도 라이브러리 없이 인라인 SVG로 S-Curve와 CPM 네트워크를 직접 그림

## 실행 / 배포 방법

이 프로젝트는 번들링이 필요 없는 정적 웹앱입니다. `package.json`은 로컬 미리보기 편의를 위한 것으로, 실제 배포 산출물은 저장소 파일 그대로입니다.

```bash
npm install     # 설치할 의존성이 없어 즉시 완료됩니다
npm run build   # 별도 번들링이 없다는 안내만 출력하고 성공 종료합니다 (CI 연동용)
npm run dev     # npx serve로 로컬 미리보기 서버를 띄웁니다 (http://localhost:5173)
```

브라우저에서 ES 모듈(`<script type="module">`)을 쓰기 때문에 `index.html`을 `file://`로 직접 열면 일부 브라우저가 모듈 로드를 막을 수 있습니다. 반드시 로컬 서버(`npm run dev`, `npx serve`, VS Code Live Server 등)로 열어주세요.

**GitHub Pages 배포**
1. 이 저장소를 GitHub에 푸시합니다.
2. 저장소 Settings → Pages → Source에서 배포 브랜치(`main`)와 루트(`/`)를 선택합니다.
3. 빌드 단계가 없으므로 별도 GitHub Actions 없이 몇 분 내 `https://<사용자명>.github.io/<저장소명>/`에서 바로 서비스됩니다.

**Vercel 배포**
1. Vercel에서 이 저장소를 Import 합니다.
2. Framework Preset을 "Other"(정적 사이트)로 두고, Build Command는 비워두거나 `npm run build`(성공만 출력) 그대로 둡니다. Output Directory는 저장소 루트(`.`)로 지정합니다.
3. Deploy를 누르면 별도 설정 없이 배포됩니다.

## 검증

- Node 스크립트로 분류→보할→기간배분→CPM→S커브 파이프라인을 단위 검증(정상 케이스, 순환참조, 공사기간 부족, 시작일>종료일 오류 케이스 포함).
- Playwright 헤드리스 브라우저로 샘플 데이터 및 실제 계층형 샘플 엑셀(`sample/샘플공종별내역.xlsx`) 업로드 → 자동분류 → 공정표 생성 → CPM 데이터/네트워크/Gantt/S-Curve 표시 → 엑셀·CSV·JSON 다운로드 → 인쇄 미리보기 → 초기화까지 전체 플로우를 콘솔 에러 없이 확인.
- CPM 네트워크 다이어그램의 Critical 노드 수가 CPM 데이터 표의 Critical(CP) 행 수와 항상 일치하는지 별도로 검증(동일한 Activity 배열을 공유하므로 구조적으로 일치).
- `npm install` / `npm run build` 정상 종료 확인.

## 알려진 제한사항 / 향후 개선사항

- 계층형 집계표에서 공종코드·상위공종 정보가 일부 누락된 경우(소계행이 코드 없이 "○"로만 표시되는 등) 자동 리크 판별이 어려울 수 있습니다. 이런 항목은 분류 후 "기타" 공종에 모이므로, ③ 탭에서 금액을 확인하고 병합해 보정하는 것을 권장합니다.
- 현재는 공종 간 선후관계를 표준 사전 기반으로 자동 결정하며, CPM 네트워크/데이터 탭은 v1 기준 "보기" 전용입니다. 추후 네트워크 다이어그램에서 노드를 드래그해 선후관계·기간을 직접 편집하면 즉시 재계산되는 기능을 추가할 예정입니다.
- 실착공정(실적) 대비 계획공정 비교, 다중 시트/다중 공사 지원, 공종별 자재/인력 투입 계획 연동은 이후 버전에서 검토합니다.
- PDF 출력은 브라우저 인쇄(다른 이름으로 저장 → PDF)를 사용합니다. 전용 PDF 렌더러(jsPDF 등) 도입은 필요 시 추가할 수 있습니다.
- 프로젝트 폴더가 클라우드 동기화 폴더(Google Drive 등) 안에 있으면 `npm install`이 파일 잠금 문제로 실패할 수 있습니다(동기화 폴더 특성상 발생하는 환경 이슈이며, 이 프로젝트는 애초에 설치할 의존성이 없어 영향은 없습니다).
